Project quality calculator

This is simple application to demonstrate how different UI patterns can be implemented using GWT.

Used patterns are:

  • MVC - in ee.urgoringo.sandbox.gwtpatterns.client.mvc package
  • Command - in ee.urgoringo.sandbox.gwtpatterns.client.command package
  • MVP (with Supervising Presenter) - in ee.urgoringo.sandbox.gwtpatterns.client.mvp package
  • MVP with Passive View - in ee.urgoringo.sandbox.gwtpatterns.client.mvp_passive_view package
  • Presentation Model - in ee.urgoringo.sandbox.gwtpatterns.client.presentationmodel package
To switch the implementation simply replace the view class in Ui_patterns_gwt.

Functionality

  • User can enter project time constraints in man-days and no of user stories that need to be implemented.
  • Based on these values application will apply "extremely complex scientific calculation" :) to figure out how many bugs you are likely going to have. Using the the number of bugs system will calculate your quality level.
  • Quality level is indicated using different font colors when displaying bug count (red, blue, black).
